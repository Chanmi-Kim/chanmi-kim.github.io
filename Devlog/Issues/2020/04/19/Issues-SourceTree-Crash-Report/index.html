[이슈해결] 소스트리 종료 현상 - SourceTree Crash Report

소스트리(Source Tree) 사용 시 종료되고 Crash Report 창이 뜨는 것에 대한 해결방법을 담았습니다.


💎 문제상황



VCS(Version Control System) 도구 중 하나인 소스트리를 자주 사용하는 편인데, 언제부턴가 계속 프로그램이 강제종료되고, Crash Report 창을 띄우기 시작했습니다. intelli J나 Visual Studio Code에 있는 VCS를 사용했었으므로 그동안 불편함을 느끼지 못하다가 최근에서야 해결방법을 찾았습니다.


제가 겪었던 경험 중 소스트리 프로그램이 강제종료되는 현상은 2가지였습니다. 새로운 노드를 생성할 때나 종종 Commit 버튼을 누를 때 프로그램이 종료되면서 다음과 같은 창이 떴습니다. 처음에는 해결방법을 몰랐기 때문에 Send Error Report 버튼만 눌렀지만 어느샌가 종료현상이 나타나지 않을 때도 있었습니다.



🔑 해결방안


해결방법은 매우 단순합니다. 사진에 있는 화살표와 빨강 네모를 참고해서 보시면 되겠습니다. 우선 기존 프로젝트가 열려있는 상태에서 오른쪽 위에 있는 설정 버튼을 누르세요. 저장소 설정에서 고급 탭 > 사용자정보 > 전역 사용자 설정 사용 체크가 되어있다면 해제하면 됩니다.


🎀 후기


정말 단순한 방법이었는데 Mac 환경에서 사용했을 때는 문제가 없었습니다. Windows에서 나타나는 현상 같습니다. 이 포스팅을 작성할 때 제 컴퓨터의 개발환경은 Windows 10입니다. Github 외에 Gitlab이나 Bitbucket 계정도 등록해서 사용 중인데, 아마 전역 사용자 설정에서 충돌이 난 것이 아닌가 추측해보고 있습니다.


혹시 다른 이유로 같은 현상이 지속된다면 업데이트해두겠습니다.
